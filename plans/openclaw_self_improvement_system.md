# Execution Plan: OpenClaw Self-Improvement System

> План создания и развития системы самосовершенствования для агента Jarviz в среде OpenClaw.  
> Цель: создать среду, в которой агент может надёжно работать, учиться на опыте и улучшаться самостоятельно.  
> Основано на принципах Harness Engineering, цикла обучения иживой документации.  
> Последняя обновление: 01.05.2026

---

## 🎯 Цель

Создать замкнутый цикл самосовершенствования:
```
Опыт работы агента
        ↓
Запись урока (что сработало/не сработало, почему)
        ↓
Сохранение знания в надёжное хранилище (vault + self-improving)
        ↓
Индексация и сделать доступным для поиска (gbrain)
        ↓
Доступность знания для будущего reasoning и принятия решений
        ↓
Применение знания в новой работе → лучшие результаты
        ↓
Новый опыт → цикл повторяется
```

Цель не просто "делать агента smarter" — а **проектировать среду**, в которой само обучение и улучшение становятся структурной частью работы.

## 📌 Принятые решения

| Решение | Обоснование | Дата |
|--------|-------------|------|
| Использовать `~/self-improving/memory.md` как append-only журнал уроков | Предотвращает потерю или перезапись прошлых уроков; простой и надёжный формат | 11.04.2026 |
| Хранить уроки в формате `* [Дата]: [Краткий урок]` | Простота, читаемость, совместимость с поиском и чтением | 11.04.2026 |
| Связать self-improving с vault через `gbrain import` | Делает уроки доступными для семантического поиска и future reasoning | 19.04.2026 |
| Добавить импорт self-improving в сессионный ритуал (AGENTS.md) | Сделать процесс автоматическим и регулярным — не relying на память | 20.04.2026 |
| Не перезаписывать `memory/YYYY-MM-DD.md`, а перезаписывать его ежедневно | Сохраняет дневной лог как свежий снимок состояния, но позволяет обновлять актуальную версию | 11.04.2026 |
| Хранить долгосрочную память в `MEMORY.md` как append-only журнал значимых выводов | Предотвращает потеря ценных идей; только добавление, никаких удалений или переформатирований | 11.04.2026 |

## ⚠️ Технический долг (Tech Debt)

| Пункт | Описание | Влияние | Приоритет | План погашения |
|-------|----------|---------|-----------|----------------|
| Отсутствие полной векторной эмбеддинга в GBrain | Из-за отсутствия OPENAI_API_KEY (или альтернативы) поиск работает только на TF-IDF и точном совпадении | Снижает качество семантического поиска — уроки находятся хуже по смыслу | Средний | Дождаться доступности API ключа или внедрить локальную эмбеддинг-модель (если ресурсы позволяют) |
| Формат уроков слишком простой | Нет структурированных полей: контекст, решение, обоснование, применение, связанные навыки | Сложно автоматически извлекать структурированную информацию для будущего использования | Средний | Ввести шаблон уроков: например, YAML-заголовок или размеченный блок с полями |
| Нет автоматической проверки актуальности ссылок в self-improving или vault | Ссылки на внешние ресурсы могут устаревать | Риск использования устаревшей информации | Низкий | Периодически (ежемесячно) проверять внешние ссылки или добавлять метаданные с датой проверки |
| Нет механизма "попадания знания в рабочий контекст" | Уроки сохраняются, но агент может не вспомнить или не проверить их перед ответом | Снижает эффективность цикла обучения | Средний | Разработать привычку: перед ответом на сложный вопрос — выполнить `gbrain search` по ключевым темам |
| Onboarding новых уроков не интегрирован с системой знаний в реальном времени | Требуется ручной импорт или ожидание цикла | Задержка между получением урока и его доступностью для поиска | Средний | Рассмотреть вебхук или триггер на файловую систему для автоматического `gbrain import` при изменении self-improving/ |

## 🔗 Связанные навыки, инструменты и документы

- Навык: `self-improving` — основной механизм записи и хранения уроков
- Навык: `gbrain` — семантический поиск и синхронизация знаний из vault и self-improving
- Навык: `skill-creator` — для создания новых навыков, которые могут улучшать самосовершенствование
- Инструмент: `web_search` — для поиска информации, которая может стать частью урока
- Инструмент: `summarize` — для суммаризации статей, видео, документов (если доступны API ключи)
- Инструмент: `read`, `write`, `exec` — для работы с файлами журнала уроков
- Связанные документы:  
  - `/root/self-improving/memory.md` — основной журнал уроков  
  - `/root/self-improving/` — доменные и проектные файлы улучшений  
  - `/root/vault/ideas/` — основное хранилище знаний (vault)  
  - `/root/.openclaw/workspace/AGENTS.md` — где реализован регулярный импорт self-improving  
  - `/root/.openclaw/workspace/MEMORY.md` — долгосрочная память значимых выводов  
  - `/root/.openclaw/workspace/SOUL.md` — кто вы есть (контекст личности и целей)  
  - `/root/.openclaw.workspace/USER.md` — о ком вы заботитесь (контекст пользователя)
