# 📘 Инструкция: Настройка RAG-системы (Knowledge Base)

Этот документ является универсальным гайдом для любого AI-агента по настройке, обслуживанию и масштабированию системы Retrieval-Augmented Generation (RAG) на базе Google Drive и локального хранилища.

---

## 1. Архитектура системы

Система состоит из трех уровней:
1.  **Storage (Хранилище):** Google Drive служит «Single Source of Truth». Структура папок определяет домены знаний.
2.  **Index (Индекс):** Локальная файловая система или векторная БД для быстрого поиска.
3.  **Agent (Агент):** ИИ, который использует `self-improving` навыки для обогащения базы.

### Структура папок на Google Drive:
```text
RAG_Knowledge_Base/
├── Agro-Meteo/          # Агрометеорология, данные с датчиков
├── Viticulture/         # Виноградарство, агрономия
├── Infrastructure/      # IT, серверы, код, промпты
├── Economics/           # Финансы, аналитика
├── Scripts/             # Утилиты для обслуживания базы
└── Instructions/        # Документация (этот файл)
```

---

## 2. Настройка окружения (Setup)

Для работы агента с базой знаний необходимы следующие компоненты:

### A. Доступ к Google Drive API
1.  Создайте проект в Google Cloud Console.
2.  Включите **Google Drive API v3**.
3.  Получите `client_id`, `client_secret` и `refresh_token`.
4.  Сохраните их в файле `google-oauth.json` в корне рабочего пространства.

### B. Локальная структура (Self-Improving)
Агент должен создать локальное зеркало для быстрой работы:
```bash
mkdir -p ~/self-improving/projects/{agro-meteo,viticulture,infrastructure,economics}
```

---

## 3. Процесс индексации и обновления

Используйте скрипт `update_rag_index.py` для синхронизации.

### Алгоритм работы скрипта:
1.  **Сканирование:** Подключение к Google Drive и рекурсивный обход папки `RAG_Knowledge_Base`.
2.  **Генерация HTML:** Создание кликабельного оглавления (`rag_index.html`).
3.  **Бинарная квантация (Опционально):** Для ускорения поиска векторы преобразуются в бинарный формат (экономия памяти до 32x).

### Команда запуска:
```bash
python3 scripts/update_rag_index.py
```

---

## 4. Оптимизация поиска: Binary Quantization

Для работы с большими объемами данных (>10,000 чанков) используйте бинарную квантацию.

**Преимущества:**
*   **Скорость:** Поиск через расстояние Хэмминга (XOR) работает в 40x быстрее косинусного поиска.
*   **Память:** Хранение в формате `int8` или битовых масках снижает затраты в 32 раза.

**Рекомендуемый стек для внедрения:**
*   `NumPy` для операций с матрицами.
*   `Milvus` или `Qdrant` (если требуется распределенный поиск).
*   `Ollama` для генерации локальных эмбеддингов.

---

## 5. Правила для AI-агента (Best Practices)

1.  **Классификация:** При получении новой информации агент должен определить её домен и сохранить в соответствующую папку.
2.  **Дедупликация:** Перед сохранением проверяй наличие похожего контента, чтобы не засорять базу.
3.  **Self-Improvement:** Если агент допускает ошибку или получает правку от пользователя, он **обязан** зафиксировать этот урок в файле `~/self-improving/corrections.md`.
4.  **Безопасность:** Никогда не сохраняй пароли, API-ключи или личные данные в открытую. Используй `.env` файлы или менеджеры секретов.
