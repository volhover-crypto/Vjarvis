# Плановые cron-задачи (ожидают активации после восстановления gateway-связи)

## 📌 Общее
Все задачи предназначены для выполнения в **изолированных сессиях** (`session: isolated`), чтобы не загрязнять основной диалог.  
Оповещения в Telegram (`announce: true`, `channel: telegram`) включены там, где это полезно для пользователя.  
Время указано в timezone `Europe/Moscow` (UTC+3), если не указано иное.

---

## 1. 📝 Дневной лог памяти
- **ID:** `daily-memory-log`
- **Расписание:** `50 23 * * *` (каждый день в 23:50 по Москве / 20:50 UTC)
- **Модель:** `claude-haiku` (быстрая, экономичная)
- **Действие:**
  - Создать файл `memory/YYYY-MM-DD.md` (где YYYY-MM-DD — сегодняшняя дата)
  - Записать структурированный итог дня:
    - `## Задачи выполнены`
    - `## Решения приняты`
    - `## Проблемы`
    - `## Открытые вопросы`
    - `## Контекст для следующей сессии`
  - Источник данных: транскрипт текущей сессии
- **Оповещение:** да — в Telegram
- **Применение (когда gateway доступен):**
  ```
  openclaw cron add \
    --name "📝 Дневной лог памяти" \
    --cron "50 23 * * *" \
    --tz "Europe/Moscow" \
    --session isolated \
    --model "claude-haiku" \
    --message "Создай файл memory/[сегодняшняя дата].md. Структура: ## Задачи выполнены / ## Решения приняты / ## Проблемы / ## Открытые вопросы / ## Контекст для следующей сессии. Читай транскрипт текущей сессии как источник данных." \
    --announce \
    --channel telegram
  ```

## 2. ☀️ Утренний брифинг
- **ID:** `morning-brief`
- **Расписание:** `0 7 * * 1-5` (каждый будний день в 07:00 по Москве / 04:00 UTC)
- **Модель:** `claude-sonnet`
- **Уровень мышления:** `medium`
- **Действие:**
  - Прочитать:
    - `memory/[вчера].md` (дневной лог за предыдущий день)
    - `MEMORY.md` (долгосрочная память)
  - Составить краткий брифинг (максимум 10 пунктов):
    - Что осталось незакрытым
    - Что запланировано сегодня
    - Какие приоритеты на день
- **Оповещение:** да — в Telegram
- **Применение:**
  ```
  openclaw cron add \
    --name "☀️ Утренний брифинг" \
    --cron "0 7 * * 1-5" \
    --tz "Europe/Moscow" \
    --session isolated \
    --model "claude-sonnet" \
    --thinking medium \
    --message "Прочитай memory/[вчера].md и MEMORY.md. Составь краткий брифинг: что осталось незакрытым, что запланировано сегодня, какие приоритеты. Максимум 10 пунктов." \
    --announce \
    --channel telegram
  ```

## 3. 📚 Недельная консолидация
- **ID:** `weekly-consolidation`
- **Расписание:** `0 4 * * 1` (каждый понедельник в 04:00 по Москве / 01:00 UTC)
- **Модель:** `claude-sonnet`
- **Уровень мышления:** `high`
- **Действие:**
  - Прочитать все файлы `memory/*.md` за последние 7 дней
  - Синтезировать ключевые инсайты
  - Добавить раздел `## Неделя [дата]` в `MEMORY.md` (например, `## Неделя 2026-04-06 -- 2026-04-12`)
  - Удалить дублирующиеся записи из дневных логов (`memory/*.md`)
  - **Важно:** Не перезаписывать `MEMORY.md` целиком — только **добавлять** новые секции и чистить логи
- **Лимит запросов:** 20 (чтобы избежать превышения лимитов при больших объёмах)
- **Оповещение:** не объявлено (по умолчанию — нет)
- **Применение:**
  ```
  openclaw cron add \
    --name "📚 Недельная консолидация" \
    --cron "0 4 * * 1" \
    --tz "Europe/Moscow" \
    --session isolated \
    --model "claude-sonnet" \
    --thinking high \
    --message "Прочитай все файлы memory/*.md за последние 7 дней. Синтезируй ключевые инсайты и добавь раздел '## Неделя [дата]' в MEMORY.md. Удали дублирующиеся записи из дневных логов. Не перезаписывай MEMORY.md — только ДОБАВЛЯЙ." \
    --requests-limit 20
  ```

## 4. 🗄️ Месячный архив
- **ID:** `monthly-archive`
- **Расписание:** `0 3 1 * *` (1-го числа каждого месяца в 03:00 по Москве / 00:00 UTC)
- **Модель:** `claude-opus` (самая мощная)
- **Уровень мышления:** `high`
- **Действие:**
  - Создать архивный файл: `memory/archive/[год]-[месяц].md` (например, `memory/archive/2026-04.md`)
  - В нём: полный дайджест прошлого месяца (события, решения, инсайты, метрики)
  - Обновить раздел `Долгосрочные паттерны` в `MEMORY.md`
  - Сделать `git commit` архива (предполагается, что репозиторий инициализирован)
- **Лимит запросов:** 30
- **Оповещение:** да — в Telegram
- **Применение:**
  ```
  openclaw cron add \
    --name "🗄️ Месячный архив" \
    --cron "0 3 1 * * " \
    --tz "Europe/Moscow" \
    --session isolated \
    --model "claude-opus" \
    --thinking high \
    --message "Создай архивный файл memory/archive/[год]-[месяц].md с полным дайджестом прошлого месяца. Обнови раздел 'Долгосрочные паттерны' в MEMORY.md. Сделай git commit архива." \
    --requests-limit 30 \
    --announce \
    --channel telegram
  ```

## 5. 🌿 Мониторинг сенсоров АСУ ТП
- **ID:** `sensor-anomaly-check`
- **Расписание:** `every 14400000 ms` (каждые 4 часа)
- **Модель:** `claude-haiku` (лёгкая, для частых проверок)
- **Действие:**
  - Прочитать файл `~/data/raw/sensors_latest.json`
  - Проверить значения:
    - pH: норма — 5.5–7.5 → тревога, если < 5.5 или > 7.5
    - Влажность: норма — 20%–90% → тревога, если < 20% или > 90%
  - Если аномалия обнаружена:
    - Отправить алерт в Telegram
    - Записать результат проверки (включая timestamp и значения) в `~/logs/sensor_checks.log`
- **Лимит запросов:** 5 (очень низкий — защита от ложных срабатываний и спама)
- **Оповещение:** не указано — по умолчанию, вероятно, нет (логируем локально, алерт только при срабатывании)
- **Применение:**
  ```
  openclaw cron add \
    --name "🌿 Мониторинг сенсоров АСУ ТП" \
    --every 14400000 \
    --session isolated \
    --model "claude-haiku" \
    --message "Проверь ~/data/raw/sensors_latest.json. Если обнаружишь значения вне нормы (pH < 5.5 или > 7.5, влажность < 20% или > 90%) — отправь алерт в Telegram. Запиши результат проверки в ~/logs/sensor_checks.log" \
    --requests-limit 5
  ```

## 🔐 Статус и требования к активации
Все задачи **ожидают активации** из-за текущей ошибки:
```
gateway connect failed: GatewayClientRequestError: pairing required
```

### Для активации необходимо:
1. **Восстановить авторизованное подключение к gateway**:
   - Убедиться, что gateway запущен: `openclaw gateway status`
   - Выполнить парынг (если требуется):
     - Через веб-дашборд: `http://127.0.0.1:18789/`
     - Или через мобильное приложение OpenClaw
     - Или обновить/повторить авторизацию токеном
   - Убедиться, что токен в `~/.openclaw/openclaw.json` раздела `gateway.auth.token` всё ещё валиден:
     ```
     "token": "037cf837b2698e910baac7a3f8d3d36d95daff5aa8b3035b"
     ```

2. **После восстановления соединения**:
   - Выполнить команды `openclaw cron add` для каждой задачи (примеры выше)
   - Либо импортировать定义 через другой способ, если появится поддержка bulk-загрузки

### 📁 Где хранятся определения:
- Точный JSON: `/root/.openclaw/workspace/cron_jobs_definition.json`
- Читаемый план: `/root/.openclaw/workspace/cron_jobs_planned.md`

## 💡 Рекомендации по использованию
- После активации рекомендуется периодически проверять список задач: `openclaw cron list`
- При неудачах — смотреть логи gateway: `/tmp/openclaw/openclaw-*.log`
- При изменении логики — обновлять определения и пересоздавать задачи (старые можно удалять через `openclaw cron rm <id>` или по jobId)
