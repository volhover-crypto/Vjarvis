# Context Engineering (OpenClaw Edition)

## Overview

Подавай агенту правильную информацию в правильное время. Контекст — главный рычаг качества вывода. Слишком мало — агент галлюцинирует. Слишком много — теряет фокус.

## Когда использовать

- Начало новой сессии
- Качество вывода падает (неправильные паттерны, галлюцинации API)
- Переключение между задачами
- Настройка нового проекта
- Агент не следует конвенциям проекта

## Иерархия контекста (OpenClaw)

```
┌─────────────────────────────────────────┐
│  1. System Prompt (OpenClaw injected)   │ ← Всегда загружен
│     AGENTS.md, SOUL.md, USER.md,        │
│     MEMORY.md, HEARTBEAT.md             │
├─────────────────────────────────────────┤
│  2. Project Context                     │ ← Загружается по проекту
│     vault/WIKI.md, vault/docs/          │
│     skills/<skill>/SKILL.md             │
├─────────────────────────────────────────┤
│  3. Task Context                        │ ← Загружается по задаче
│     vault/ideas/, vault/projects/       │
│     memory/YYYY-MM-DD.md                │
├─────────────────────────────────────────┤
│  4. Transient Context                   │ ← Текущая сессия
│     История сообщений, файлы сессии      │
└─────────────────────────────────────────┘
```

## Правила контекст-инжиниринга

### 1. Размер контекста ≈ качество до порога

```
Слишком мало → галлюцинации, игнорирование конвенций
Оптимально  → точные ответы, следование паттернам
Слишком много → потеря фокуса, повторения, медленные ответы
```

**Наша оптимальная зона:** AGENTS.md + SOUL.md + MEMORY.md + 1-2 релевантных файла из vault.

### 2. Загружай контекст прогрессивно

**НЕ загружай всё сразу.** Загружай по мере необходимости:

```
Сессия начата
    │
    ▼
AGENTS.md + SOUL.md + MEMORY.md (автоматически)
    │
    ▼
Определить домен задачи → загрузить релевантный vault/docs/
    │
    ▼
Определить навык → загрузить SKILL.md
    │
    ▼
Выполнить задачу
```

### 3. Структурируй файлы контекста

**AGENTS.md** — только навигация (≤100 строк), без знаний
**MEMORY.md** — только ключевые факты (≤100 строк), обучение вынести отдельно
**vault/docs/** — справочники по доменам, структурированные
**vault/learning-log.md** — лог обучения, хронологический

### 4. Обновляй контекст при изменениях

При любом значительном изменении:
1. Обновить релевантный файл контекста
2. Записать в vault/learning-log.md
3. Обновить MEMORY.md если изменилась конфигурация

### 5. Используй теги и метаданные

В файлах vault используй frontmatter:
```yaml
---
tags: [AI, ML, обучение]
date: 2026-05-11
source: https://example.com/article
related: [[другая_страница]]
---
```

Это позволяет агенту быстрее находить релевантный контекст.

## Контекст для наших доменов

### AI/ML задачи
- `vault/docs/ai_tools/` — справочники по инструментам
- `vault/knowledge/` — структурированные знания
- `skills/agent-skills/` — инженерные навыки

### Агрономия/IoT
- `vault/docs/agronomy/` — агрономические справочники
- `vault/docs/iot/` — IoT и сенсоры
- `vault/projects/` — проектная документация

### Системное администрирование
- `skills/system-resource-monitor/` — мониторинг
- `skills/healthcheck/` — проверка здоровья
- `TOOLS.md` — заметки по инструментам

## Признаки плохого контекста

- Агент галлюцинирует API/функции
- Агент игнорирует установленные конвенции
- Агент повторяется или теряет фокус
- Агент не помнит предыдущие решения
- Агент даёт слишком общие ответы

## Решение проблем контекста

| Проблема | Решение |
|----------|---------|
| Галлюцинации API | Загрузить справочник по инструменту |
| Игнорирование конвенций | Обновить AGENTS.md / SOUL.md |
| Потеря фокуса | Уменьшить контекст, сфокусировать на задаче |
| Забывает решения | Записать в vault/learning-log.md |
| Слишком общие ответы | Загрузить конкретный справочник по домену |

## Checklist

- [ ] AGENTS.md компактный (≤100 строк), только навигация
- [ ] MEMORY.md компактный (≤100 строк), только ключевые факты
- [ ] Обучение вынесено в vault/learning-log.md
- [ ] Файлы vault имеют frontmatter с тегами
- [ ] Контекст загружается прогрессивно, не всё сразу
- [ ] При изменениях контекст обновляется
