# Dreaming — Анализ прошлых сессий

## Концепция

Аналог функции «Dreaming» от Anthropic. Периодически анализирует прошлые сессии и дневные логи для:
- Выявления повторяющихся ошибок и паттернов
- Извлечения ключевых инсайтов
- Обновления принципов работы
- Эскалации нерешённых проблем

## Когда запускать

1. **Еженедельно** (понедельник 09:00 MSK) — анализ последних 5 дневных логов
2. **Ежемесячно** (1-го числа 10:00 MSK) — глубокий анализ всех логов за месяц
3. **По триггеру** — после завершения крупного проекта или при обнаружении системной проблемы

## Процесс

### Еженедельный анализ (лёгкий)

1. Прочитать последние 5 файлов из `memory/` (дневные логи `YYYY-MM-DD.md`)
2. Для каждого лога выделить:
   - Какие задачи выполнялись?
   - Какие ошибки возникли?
   - Какие решения были найдены?
   - Что осталось нерешённым?
3. Найти повторяющиеся паттерны (≥2 раза)
4. Записать 2-3 ключевых инсайта в `MEMORY.md` с тегом `[DREAMING]`
5. Если найден новый принцип — обновить `memory/principles_lib.md`

### Ежемесячный анализ (глубокий)

1. Прочитать ВСЕ дневные логи за месяц
2. Сформировать отчёт «Паттерны месяца»:
   - Какие задачи повторялись чаще всего?
   - Где были типичные ошибки?
   - Какие решения оказались наиболее эффективными?
   - Какие инструменты/подходы использовались чаще?
3. Записать отчёт в `MEMORY.md`
4. Обновить `memory/principles_lib.md`
5. Проверить: какие нерешённые проблемы висят >1 месяца? → эскалировать

### Формат записи в MEMORY.md

```
## DREAMING [дата]
### Паттерны:
- [паттерн 1]: [описание] → [вывод]
- [паттерн 2]: [описание] → [вывод]
### Инсайты:
- [инсайт 1]
- [инсайт 2]
### Нерешённые проблемы:
- [проблема]: [статус], [рекомендация]
```

## Интеграция с другими навыками

- **Self-Improving** — Dreaming дополняет самообучение, добавляя периодический ретроспективный анализ
- **HEARTBEAT.md** — задачи Dreaming встроены в еженедельные и ежемесячные проверки
- **memory/principles_lib.md** — найденные принципы записываются в библиотеку
- **GBrain** — при наличии индекса, использовать семантический поиск по логам для поиска паттернов

## Примеры паттернов для поиска

- Ошибки форматирования (например, неправильный порядок полей в KiCad PCB)
- Повторяющиеся проблемы с инструментами (например, exec preflight блокировка)
- Типичные запросы пользователя
- Эффективные решения для повторного использования
- Потери контекста при длинных сессиях

## Настройка

Задачи Dreaming автоматически запускаются через HEARTBEAT.md:
- Еженедельно: понедельник 09:00 MSK
- Ежемесячно: 1-го числа 10:00 MSK

Для ручного запуска: «проведи анализ прошлых сессий» или «dreaming».
