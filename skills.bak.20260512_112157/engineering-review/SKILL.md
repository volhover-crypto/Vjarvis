# Engineering Review — Рецензирование технических проектов

## Когда использовать
Анализ проектной документации, проверка инженерных расчётов, рецензирование чертежей, оценка технических решений, выявление ошибок и несоответствий стандартам.

## Область применения

Механика, гидравлика, электротехника, строительные конструкции, трубопроводы, системы автоматизации, АСУ ТП.

## Этапы рецензирования

### 1. Формальная проверка
- Полнота комплекта документации
- Соответствие ГОСТ/СП/СНиП на оформление
- Наличие всех разделов: пояснительная записка, расчёты, чертежи, спецификации
- Корректность ссылок на нормативные документы

### 2. Проверка расчётов
- Размерность — все единицы СИ, проверка перевода единиц
- Формулы — корректность применения, соответствие условиям
- Числовые значения — подстановка, арифметика
- Запасы прочности — соответствие нормативным (обычно 1.2–2.0)
- Граничные условия — проверка на предельных значениях

### 3. Анализ конструктивных решений
- Работоспособность — выполняет ли конструкция поставленную задачу
- Технологичность — возможно ли изготовление существующими методами
- Стандартизация — применение стандартных деталей и материалов
- Ремонтопригодность — возможность замены и обслуживания
- Безопасность — наличие защитных мер, блокировок, аварийных режимов

### 4. Проверка чертежей
- Соответствие ЕСКД (ГОСТ 2.101–2.702)
- Полнота и правильность простановки размеров
- Допуски и посадки — обоснованность, возможность контроля
- Материалы — корректность указания марок и ГОСТ
- Технологические указания — обработка, покрытие, термообработка

### 5. Проверка на соответствие стандартам

| Область | Ключевые стандарты |
|---------|-------------------|
| Оформление | ГОСТ 2.105-2019, ГОСТ 2.301-68 |
| Чертежи | ГОСТ 2.104-2006, ГОСТ 2.307-2011 |
| Расчёты прочности | ГОСТ 27701-88, СП 16.13330 |
| Трубопроводы | ГОСТ 14202-69, СП 36.13330 |
| Электрика | ГОСТ 21.613-2014, ПУЭ |
| АСУ ТП | ГОСТ 34.201-89, ГОСТ 34.601-90 |
| Пожарная безопасность | СП 12.13130, ФЗ-123 |

## Формат выходного отчёта

```
# Акт рецензии: [наименование проекта/документа]

## Общая информация
- Рецензент: [роль]
- Дата: [дата]
- Объект: [что рецензируется]

## Резюме
Краткое заключение: допустимо / допустимо с замечаниями / требует переработки

## Замечания

### Критические (требуют обязательного устранения)
1. [Описание, ссылка на лист/раздел, пояснение]

### Существенные (рекомендуется устранить)
1. [Описание, ссылка, пояснение]

### Замечания по оформлению
1. [Описание]

## Итоговая оценка
[Число] замечаний, в т.ч. [N] критических.

Документация [соответствует/не соответствует] требованиям [стандарт].
```

## Тон и стиль

- Фактологичный, без эмоций
- Ссылки на конкретные пункты стандартов
- Разделение на критические, существенные и оформительские замечания
- Конкретные рекомендации по устранению, не просто «неверно»
