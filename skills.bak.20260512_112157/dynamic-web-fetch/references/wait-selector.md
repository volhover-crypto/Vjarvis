# Ожидание селектора

При загрузке динамической страницы часто необходимо дождаться появления определенного элемента в DOM перед извлечением контента.

## Использование

Навык поддерживает ожидание селектора через CSS или XPath селекторы.

### Параметры ожидания

- `selector`: CSS селектор для ожидания элемента (например, `.sorts-list`, `#content`, `div.article`)
- `xpath`: XPath селектор как альтернатива CSS
- `timeout`: Время ожидания в миллисекундах (по умолчанию 30000 мс)
- `state`: Состояние элемента для ожидания:
  - `attached`: элемент присутствует в DOM
  - `visible`: элемент видим и не имеет `display: none` или `visibility: hidden`
  - `hidden`: элемент скрыт (противоположность visible)
  - `enabled`: элемент активен (для формовых элементов)

### Примеры использования

```javascript
// Ожидание появления списка сортировки
await page.waitForSelector('.sorts-list', { state: 'visible' });

// Ожидание загрузки основного контента
await page.waitForSelector('#main-content', { state: 'attached', timeout: 15000 });

// Ожидание исчезновения индикатора загрузки
await page.waitForSelector('.loading-spinner', { state: 'hidden' });
```

### Лучшие практики

1. Используйте специфичные селекторы чтобы избежать ложных срабатываний
2. Предпочитайте CSS селекторы перед XPath для лучшей производительности
3. Устанавливайте разумные таймауты чтобы избежать бесконечного ожидания
4. Для AJAX-загрузки контента комбинируйте с ожиданием сети
