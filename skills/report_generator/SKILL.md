# Report Generator — Генератор отчётов

## Когда использовать
Запросы на формирование отчётов, сводок, дашбордов по данным мониторинга, агрономическим наблюдениям, результатам анализов.

## Типы отчётов

### 1. Суточная сводка метео
- Мин/макс/средняя температура
- Сумма осадков
- Средняя влажность
- Скорость ветра
- GDD за день (если есть данные)

### 2. Недельный отчёт по винограднику
- Фенофаза (BBCH)
- Накопленный GDD
- Состояние почвы (влажность, EC, pH)
- Риск болезней (по погодным условиям)
- Рекомендации по поливу/защите

### 3. Отчёт по сенсорам
- Статус всех датчиков (online/offline)
- Временные ряды показаний
- Инциденты (выход за пороги, зависания)
- Качество данных (% пропусков)

### 4. Сезонный отчёт
- Суммарный GDD за сезон
- Динамика NDVI
- Урожайность по участкам
- Сравнение с прошлыми сезонами

## Формат вывода

### Telegram (основной)
- Краткий, структурированный
- Эмодзи для быстрого сканирования
- Ключевые цифры выделены
- Рекомендации в конце

### Markdown (для Vault)
- Полный формат с таблицами
- Ссылки на источники данных
- Графики (если сгенерированы)

### PDF (для архива)
- Через Python (matplotlib + reportlab)
- Для официальных отчётов

## Шаблон суточной сводки
```
🌤 Сводка за [дата]

🌡 Температура: [min]°C / [avg]°C / [max]°C
💧 Влажность: [min]% / [avg]% / [max]%
🌧 Оадки: [sum] мм
💨 Ветер: [avg] м/с, [direction]

🍇 Виноградник:
- Фенофаза: BBCH [code]
- GDD сегодня: [value] (всего: [total])
- Влажность почвы: [value]% ([status])

⚠️ Алерты: [список или "нет"]
📋 Рекомендации: [список]
```

### 5. Отчёт по работе (what-did-i-get-done)

**Цель:** краткая сводка выполненной работы за период.

**Триггер:** «что было сделано», «сводка за неделю», «итоги», «what did I get done»

**Workflow:**
1. Определить временное окно (по умолчанию — последние 7 дней)
2. Собрать коммиты из всех репозиториев (`git log --since="..." --oneline`)
3. Исключить merge-коммиты и косметические изменения
4. Синтезировать ключевые изменения в концизный статус

**Формат вывода:**
```
📊 Отчёт по работе: [дата начала] — [дата окончания]

<1-2 предложения: главное что было сделано>

Ключевые изменения:
• [<repo>] <существенное изменение>
• [<repo>] <существенное изменение>

Всего коммитов: N | Файлов изменено: M
```

**Guardrails:**
- Экстремально концизно, информационно плотно
- Приоритет — поведенческие и архитектурные изменения
- Косметику (форматирование, импорты, переименования) опускать
- Не выдумывать мотивацию — описывать функционально
