# IoT Monitor — Мониторинг сенсоров

## Когда использовать
Вопросы о состоянии датчиков, показаниях сенсоров, настройке мониторинга, интеграции данных с датчиков, пороговых значениях для оповещений.

## Типы сенсоров

### Почвенные
- **Влажность почвы**: ёмкостные (%), тензиометры (кПа)
- **EC (электропроводность)**: 0.5–4.0 мСм/см (норма для винограда <2.5)
- **pH**: 5.5–7.5 (норма для винограда 6.0–7.0)
- **Температура почвы**: 10–25°C (оптимум корневой зоны)

### Метео
- **Температура воздуха**: датчик T/RH
- **Влажность воздуха**: 40–80% (критично для милдью при >90%)
- **Оадки**: радарные датчики (SEMR226, RK400-13)
- **Ветер**: анемометр (скорость, направление)
- **Солнечная радиация**: пиранометр (W/m²)

### Растительные
- **NDVI**: 0.2–0.6 (зависит от фенофазы)
- **Температура листа**: ИК-датчик
- **Диаметр ягоды**: линейный датчик (для мониторинга созревания)

## Протоколы и интеграция
- **MQTT**: основной протокол для IoT-датчиков
- **Modbus RTU/TCP**: промышленные датчики
- **n8n**: автоматизация обработки данных
- **PostgreSQL + TimescaleDB**: хранение временных рядов

## Пороговые значения для алертов

| Параметр | Норма | Предупреждение | Критично |
|----------|-------|----------------|----------|
| Влажность почвы | 40–60% | <30% или >70% | <20% или >80% |
| EC почвы | <2.5 мСм/см | 2.5–3.5 | >3.5 |
| pH почвы | 6.0–7.0 | 5.5–6.0 или 7.0–7.5 | <5.5 или >7.5 |
| T воздуха | 15–30°C | <10°C или >35°C | <5°C или >40°C |
| RH воздуха | 40–80% | >85% | >90% (риск милдью) |

## Алгоритм мониторинга
1. Определи доступные датчики и их протоколы
2. Настрой сбор данных (опрос по расписанию или MQTT)
3. Валидируй данные (выбросы, нули, зависания)
4. Сравни с порогами
5. При отклонении — алерт + рекомендация
