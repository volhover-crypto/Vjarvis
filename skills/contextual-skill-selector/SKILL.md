# Навык: Контекстный выбор навыков (Contextual Skill Selector)

## Описание
Навык, который анализирует описание задачи и возвращает список рекомендуемых навыков OpenClaw с обоснованием выбора, основанным на:
- ключевых словах в запросе,
- доменах экспертизы (из USER.md),
- принципах работы (из AGENTS.md, principles.md),
- структуре доступных навыков и знания (из skills/, docs/).

Предназначен для использования:
- перед сложной задачей (планирование: «какие навыки могут быть полезны?»),
- в цикле рефлексии (было ли использовано всё необходимое?),
- в мета-обучении (какие навыки полезны в каких контекстах?),
- и в интеграции с TaskFlow (пометка задач требуемыми навыками).

## Как использовать
Вызовите навык с описанием задачи в поле `task`:
```
/skills run contextual-skill-selector --task "проанализировать документ по модели заморозков и построить собственный прогноз на Python"
```

Навык вернёт список рекомендуемых навыков с обоснованием, например:
- `pdf` — для чтения документа и изучения архитектуры модели Frost LSTM
- `python` — для реализации собственного прогноза на основе LSTM или регрессии
- `self-improving` — для рефлексии по качеству модели и извлечения урока
- `weather` — для получения актуальных метеоданных (если нужно обогатить входные признаки)
- `arxiv` — для поиска современных подходов к прогнозу температуры
- `Aider` — для совместного написания и отладки кода

## Возвращаемый формат
Навык возвращает plain text или markdown с разметкой:
```
🔍 Задача: [описание задачи]

✅ Рекомендуемые навыки:
1. <навык> — <обоснование>
2. <навык> — <обоснование>
...
```

## Принципы работы
- Использует ключевые слова, домены и типы задач для сопоставления с навыками.
- Учитывает релевантность, доменную соответственность и универсальную полезность (например, `self-improving` почти всегда полезен).
- Не требует внешних API — работает исключительно на основе локальных данных: `skills/`, `docs/`, `USER.md`, `AGENTS.md`, `principles.md`.
- Безопасен: не выполняет внешних запросов, не модифицирует систему, только анализирует и рекомендует.

## Технические детали
- Реализован как простая логика сопоставления ключевых слов → навыков с весами.
- Может быть расширен за счёт добавления правил в комментариях или внешнего конфига (например, YAML-правила).
- Не требует сложных зависимостей — работает в среде OpenClaw на Node.js.
