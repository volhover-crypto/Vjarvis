# Навык: quick-check

**Назначение**:  
Быстрая проверка статуса системы и ключевых сервисов. Возвращает краткий отчёт о времени, использовании ресурсов и сетевом доступе — полезен для старта сессии, самодиагностики и ритуалов готовности к работе.

**Как использовать**:  
Спросите:  
- «Выполни быструю проверку системы»  
- «Как статус системы?»  
- «Проверить ресурсы и сеть»  
- «Quick check»  

**Возвращает**:  
Текстовый отчёт с разделами:
- 🕒 Текущее время и дата
- 💻 Использование ресурсов (CPU, RAM, диск)
- 🌐 Статус сетевого подключения
- 🔌 Состояние ключевых сервисов (при наличии проверок)

**Примеры использования**:
- `quick-check`  
- `проверить систему`  
- `статус ресурсов`  

**Сценарии применения**:
- Перед началом аналитической сессии — убедиться, что система готова
- После внесения изменений в настройки — проверить, всё ли работает
- Как часть ритуала начала работы или heartbeat-чека
- Для быстрой диагностики при подозрении на замедление или сбои

**Как это работает**:
Навык выполняет последовательность лёгких системных проверок:
1. Получает текущее время и дату через `date`
2. Проверяет использование ресурсов (если доступен навык `system-resource-monitor` — использует его, иначе — через `top`, `free`, `df`)
3. Проверяет доступ в интернет (через `ping -c 1 1.1.1.1` или `curl -s --max-time 5 https://httpbin.org/ip`)
4. Проверяет статус ключевых сервисов (GBrain, Vikunja, Tailscale и т.д.) — если они настроены и доступны
5. Формирует структурированный, читаемый отчёт

**Возвращаемый формат**:
```
🕒 Текущее время: 30.04.2026 01:45 GMT+5

💻 Ресурсы:
- CPU: 12%
- RAM: 3.2 ГБ / 7.8 ГБ (41%)
- Диск: 18 ГБ / 120 ГБ (15%)

🌐 Сеть:
- Интернет: ✅ Доступен (ответ за 12мс)
- Трейс до 1.1.1.1: OK

🔌 Сервисы:
- GBrain: ✅ Доступен
- Vikunja: ✅ Доступен (http://localhost:3456)
- Tailscale: ✅ Активен (статус: login)
- ProtonMail: ✅ Доступен через веб-интерфейс
```

**Ограничения**:
- Не заменяет глубокий мониторинг или специализированные инструменты
- Проверка сервисов зависит от их доступности по стандартным портам и URL
- Для точных метрик ресурсов рекомендуется использовать навык `system-resource-monitor`
- Некоторые проверки требуют доступа к локальным сервисам или сети

**Требования**:
- Навык работает в стандартном Linux-окружении
- Требует доступ к командам: `date`, `top`, `free`, `df`, `ping`, `curl` (если доступны)
- Для оптимальной работы рекомендуется иметь навык `system-resource-monitor` установленным
