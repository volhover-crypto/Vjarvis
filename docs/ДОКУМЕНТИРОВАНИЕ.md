## Правило документирования

Каждый рабочий процесс, хук, скрипт, настройка — должен быть задокументирован в текстовую инструкцию и сохранён в:

1. **workspace/docs/** — для внутреннего использования
2. **vault/projects/** — для Obsidian (зеркало)

Формат: markdown с заголовками, примерами и командами.

Это гарантирует что после перезапуска сессии или потери контекста — любой процесс можно восстановить по документации.
