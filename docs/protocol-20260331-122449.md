## 1. Основной проект: база данных + N8N

**Контекст:** Создаётся база данных для клиента. Инфраструктура будет работать через N8N.

**Ключевые решения:**
- База делается **для клиента** («у них»), не для себя
- Наполнение и извлечение данных — через **N8N API**
- Старые сценарии и исторические данные будут загружены в базу
- Если потребуется — добавят Kubernetes («киперун»)
- Удалённый доступ — через VPN + RDP

## 2. Метеоданные и моделирование

**Параметры:**
- Прямые: солнечная радиация и др. с метеостанции
- Производные: ГДИ, сумма активных температур
- Дни физического отбора — временная лента с датами

**Подход:**
- Формула уже выведена, конспект имеется
- Период анализа: ~6 месяцев до сбора урожая
- Модель предсказывает параметры (например, сахар) на конкретных участках
- Если предсказание не совпадает — «напрягают нейронку» для коррекции
