# memory/principles_lib.md — Библиотека стратегических принципов (EvolveR)

> Хранит абстрактные стратегические принципы, извлечённые из опыта агента через процесс offline self-distillation (EvolveR).  
> Каждый принцип имеет:
> - Текст принципа
> - Метрика эффективности (applied_count, success_rate, last_applied)
> - Источник (из какой записи/опыта получен)
> - Статус (active, deprecated, under_review)
>  
> Формат основан на EvolveR:  
> - Семантическая дедупликация предотвращает взрывной рост дублирующих принципов  
> - Интеграция позволяет новым принципам обновлять или заменять существующие  
> - Метрика эффективности ранжирует принципы по историческому успеху  
>  
> Принципы с success_rate < 0.4 и applied_count > 5 → предлагать пересмотр  
>  
> Используется в фазе Online Interaction: агент может вызвать `<search_experience>` для получения релевантных принципов перед рассуждением

### [2026-05-19] Принципы работы с GBrain на ограниченных ресурсах

**Контекст:** GBrain v0.35.8 на сервере с 1.8GB RAM

**Принципы:**
1. **AI Gateway — роскошь, а не необходимость.** Keyword search (tsvector) и hybrid search (tsvector + pgvector) работают без AI gateway. Эмбеддинги уже сгенерированы (1579 из 1840 чанков). Для поиска по существующей базе API ключ не нужен.

2. **RAM-патч — необратимое изменение.** Патч в cli.ts (configureGateway только при наличии ключа) должен применяться при КАЖДОЙ пересборке gbrain. Забыть = получить OOM killer.

3. **GBrain Lite — страховка.** Лёгкая обёртка через psql должна всегда быть доступна как fallback. Если gbrain падает — gbrain-lite работает.

4. **Embedding — отдельная задача.** Генерация новых эмбеддингов требует либо API ключа, либо локальной Ollama-модели. На текущем сервере — только с API ключом и при свободной RAM.

5. **Исходники священны.** /tmp/gbrain-src/ — НЕ УДАЛЯТЬ. Без них пересборка с RAM-патчем невозможна.

**Источник:** Практический опыт восстановления gbrain после удаления при очистке диска 18.05.2026
