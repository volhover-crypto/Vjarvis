## Цепи Маркова в предсказании — структурированный обзор

---

### 📌 Суть метода

Цепь Маркова — это стохастическая модель, где **вероятность следующего состояния зависит только от текущего** (свойство Маркова), а не от всей предыстории. Математически:

```
P(X_{n+1} = j | X_n = i, X_{n-1}, ..., X_1) = P(X_{n+1} = j | X_n = i)
```

Ключевой объект — **матрица переходных вероятностей** P, где P[i][j] = вероятность перехода из состояния i в состояние j. Сумма по каждой строке = 1.

---

### 🔧 Как работает предсказание

1. **Определить пространство состояний** (например: ясно / облачно / дождь)
2. **Оценить матрицу переходов** из исторических данных (частоты переходов)
3. **Прогноз на n шагов вперёд**: умножить вектор текущих вероятностей на P^n
4. **Стационарное раределение** π (πP = π) — долгосрочное поведение системы

---

### 🌐 Области применения для прогнозирования

| Область | Что предсказывают | Пример |
|---------|-------------------|--------|
| **Погода** | Состояние на завтра/неделю | Матрица переходов «ясно→дождь» и т.д. |
| **Финансы** | Рыночные режимы, цены | Переходы «бычий→медвежий» рынок |
| **NLP** | Следующее слово/символ | N-граммные модели — частный случай цепей Маркова |
| **PageRank** | Ранжирование страниц | Google моделирует случайное блуждание пользователя как цепь Маркова |
| **Речь** | Последовательности фонем | Hidden Markov Models (HMM) в ASR |
| **Предиктивное обслуживание** | Отказ оборудования | Переходы «норма→деградация→отказ» |
| **Климатология** | Моделирование изменений климата | ScienceDirect: интеграция с физическими моделями |
| **Генерация контента** | Текст, музыка, код | Последовательности на основе переходов из корпуса |

---

### ⚡ Сильные стороны

- **Простота**: легко интерпретировать, быстро считается
- **Мало памяти**: не нужна вся история — только текущее состояние
- **Стационарное распределение**: позволяет анализировать долгосрочное поведение
- **Эргодичность**: при выполнении условий — сходится к единственному равновесию

### ⚠️ Ограничения

- **Марковость редко точна на практике**: реальные процессы часто зависят от более глубокой истории
- **Дискретизация состояний**: непрерывные данные нужно квантовать — теряем информацию
- **Порядок**: цепь 1-го порядка не ловит зависимости вида «если было A, потом B, то скорее C» → выход: **цепи высших порядков** или **Hidden Markov Models**
- **Нестационарность**: если вероятности переходов меняются со временем, модель устаревает

### 🔗 Связь с твоими проектами

Для **виноградников в Крыму** цепи Маркова могут применяться для:
- **Прогноз фенофаз**: переходы между стадиями BBCH (распускание почек → цветение → созревание)
- **Модель рисков болезней**: переходы «здоров → инфицирован → симптомы → потери»
- **Прогноз погоды на короткий горизонт** (1-3 дня) для планирования полива/обработок
- **Анализ данных датчиков**: переходы между состояниями влажности/EC/pH

Для более точных прогнозов рядов данных с датчиков LSTM, которые ты уже используешь, — это по сути «цепь Маркова с памятью», где скрытое состояние несёт информацию о всей истории.
