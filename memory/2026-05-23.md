# 2026-05-23 — Daily Log

## n8n Auth System — Завершение (начато 22.05)

### Проблема
Вчера не удалось завершить авторизацию через n8n webhook. Code node с `require('pg')` не работал из-за sandbox изоляции n8n.

### Решение
1. **FastAPI Auth Microservice** (`/root/auth-service/main.py`):
   - FastAPI + asyncpg + bcrypt
   - Порт 8765, systemd unit (`jarvis-auth.service`)
   - Endpoints: `/auth/login`, `/auth/verify`, `/auth/logout`, `/health`
   - Хранение паролей через bcrypt

2. **n8n workflow** (ID: `rFNcxozuWQI5c0wo`):
   - `network_mode: host` для доступа к localhost:8765
   - Webhook (`POST /auth/login`) → HTTP Request (`POST http://localhost:8765/auth/login`) → Respond
   - Без Code node — всё через HTTP Request

3. **HTML форма** (`/root/.openclaw/workspace/web/login.html`):
   - Минималистичный дизайн, тёмная тема
   - POST к `http://176.12.74.69:5678/webhook/auth/login`
   - Сохранение токена в localStorage

### Тестирование
- ✅ Неверный пароль → `{"success":false,"error":"Invalid credentials"}`
- ✅ Верный пароль → `{"success":true,"token":"...","user":{...}}`
- ✅ Health check → `{"status":"ok","db":"connected"}`

### Уроки
- n8n Code node sandbox не позволяет `require()` внешних модулей
- n8n в bridge network не видит localhost хоста → нужен `network_mode: host` или отдельный контейнер в той же сети
- PostgreSQL слушает на 127.0.0.1 — из контейнеров не достучаться
- Лучше выносить сложную логику в отдельные микросервисы
