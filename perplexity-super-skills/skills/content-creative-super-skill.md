### 2.2 Design Philosophy Creation Workflow

Before producing any visual output, establish a **Visual Philosophy** — an aesthetic movement that guides all creative decisions.

#### Step 1: Name the Movement (1–2 words)

Examples: "Brutalist Joy" / "Chromatic Silence" / "Metabolist Dreams" / "Concrete Poetry" / "Analog Meditation"

#### Step 2: Write the Philosophy Manifesto (4–6 paragraphs)

Address each dimension once, without repetition:

- **Form and space**: How do forms relate to negative space? Monumental or intimate?
- **Color and material**: What palette governs? What textures or material qualities?
- **Scale and rhythm**: What is the relationship of large to small, fast to slow?
- **Composition and balance**: Symmetry, asymmetry, diagonal flow, grid-breaking?
- **Visual hierarchy**: Where does the eye go first, second, last?

**Critical requirement:** Emphasize craftsmanship repeatedly — the final work must look "meticulously crafted," "the product of deep expertise," "painstaking attention," "master-level execution."

#### Step 3: Identify the Conceptual Seed

Extract the subtle, niche reference embedded in the user's request. This is the soul woven invisibly into the art — like a jazz musician quoting another song. Only those who know will catch it; everyone else experiences a masterful composition.

#### Step 4: Execute on Canvas

Rules for canvas execution:
- 90% visual, 10% essential text (text as visual accent, not explanation)
- Nothing overlaps or falls off canvas boundaries — proper margins always
- Typography is design-forward: search `./canvas-fonts` for unique fonts; pair a distinctive display font with a refined body font
- Never use generic fonts (Arial, Inter, Roboto, system fonts)
- Aim for work that looks like it took countless hours by someone at the top of their field
- For refinement passes: make existing elements more cohesive rather than adding new elements

#### Step 5: Refinement Pass

When refining, ask: "How can I make what's already here more of a piece of art?" Avoid adding more shapes or graphics. Instead: tighten spacing, improve color harmony, strengthen typographic choices, deepen compositional balance.

### 2.4 Design Principles Quick Reference

#### Color Theory
- **Analogous**: Adjacent on color wheel — harmonious, low contrast (e.g., blue + blue-green + teal)
- **Complementary**: Opposite — high contrast, vibrant tension (e.g., orange + blue)
- **Triadic**: Three evenly spaced — balanced, energetic (e.g., red + yellow + blue)
- **Split-complementary**: One color + two adjacent to its complement — softer than complementary
- **Monochromatic**: One hue, varying saturation/value — sophisticated, cohesive

#### Dominant Color Systems
- Set a **dominant color** (60%), a **secondary color** (30%), and an **accent** (10%)
- Dominant colors with sharp accents outperform timid, evenly-distributed palettes
- Use CSS custom properties / design tokens for consistency across components

#### Typography Pairing Rules
- Pair a distinctive **display font** (headings, large text) with a refined **body font** (readable at small sizes)
- Never use: Inter, Roboto, Arial as primary display fonts
- Strong pairing examples: Playfair Display + Source Serif Pro / Clash Grotesk + Instrument Serif / Bebas Neue + DM Sans
- Weight contrast: bold display vs. regular body creates visual hierarchy without color

#### Composition Principles
- **Rule of thirds**: Place key elements at grid intersections
- **Visual weight**: Larger, darker, and more saturated elements carry more weight
- **Negative space**: Intentional emptiness creates breathing room and focus
- **Alignment**: Every element should align to a grid or another element — never float arbitrarily
- **Hierarchy**: One dominant element, one secondary, everything else subordinate

### 3.5 Language System

#### Words to Use (by archetype)
- **Action verbs**: Transform, accelerate, optimize, unlock, elevate, pioneer, craft, build, shape
- **Positive descriptors**: Seamless, powerful, intuitive, strategic, intentional, thoughtful
- **Outcome-focused**: Results, growth, impact, clarity, momentum, trust, confidence

#### Words to Avoid (universal)
- **Jargon**: Synergy, leverage (as verb), bandwidth (for availability), circle back, move the needle
- **Overused claims**: Innovative, disruptive, cutting-edge (unless genuinely applicable)
- **Weak qualifiers**: Very, really, just, maybe, hopefully, kind of, sort of
- **Negative framing**: Can't, won't, impossible — replace with challenge, opportunity, working on

### 4.2 Design Thinking Before Coding

Before writing a single line of code, answer:

1. **Purpose**: What problem does this interface solve? Who uses it?
2. **Tone**: Commit to an aesthetic extreme — don't default to "clean and modern"
   - Brutally minimal / Maximalist chaos / Retro-futuristic / Organic/natural
   - Luxury/refined / Playful/toy-like / Editorial/magazine / Brutalist/raw
   - Art deco/geometric / Soft/pastel / Industrial/utilitarian / Cyberpunk
3. **Differentiation**: What is the one thing someone will remember?
4. **Typography anchor**: Which font pair defines the visual identity?

**Critical rule:** Bold maximalism and refined minimalism both succeed — what fails is timidity. Execute your chosen aesthetic with full commitment and precision.

### 4.3 Frontend Aesthetics Guidelines

#### Typography
- Choose fonts that are **beautiful, unique, and interesting**
- Never default to: Inter, Roboto, Arial, system-ui as display fonts
- Strong display choices: Clash Grotesk, Playfair Display, Bebas Neue, Canela, Fraunces, Syne, General Sans
- Strong body choices: Source Serif Pro, DM Sans, Instrument Sans, Literata, Spectral
- Pair a distinctive display font with a refined, readable body font

#### Color & Theme
- Commit to a **cohesive aesthetic** — use CSS custom properties for consistency
- Set a dominant color (60%), secondary (30%), accent (10%)
- Dominant + sharp accent outperforms timid, evenly distributed palettes
- Never default to purple gradients on white backgrounds

#### Motion & Animation
- CSS-only for HTML projects; Motion library for React
- **One well-orchestrated page load** with staggered reveals creates more delight than scattered micro-interactions
- Use `animation-delay` for staggered entry sequences
- Scroll-triggering and hover states that genuinely surprise
- Avoid: janky transitions, gratuitous spinning, motion for motion's sake

#### Spatial Composition
- Unexpected layouts: asymmetry, overlap, diagonal flow, grid-breaking elements
- Generous negative space **or** controlled density — commit to one
- No default centered-column layouts unless the content demands it

#### Backgrounds & Visual Details
- Create atmosphere with: gradient meshes, noise textures, geometric patterns, layered transparencies
- Dramatic shadows, decorative borders, custom cursors, grain overlays when appropriate to the aesthetic
- Never default to `background: #ffffff` without intentional reasoning

### 4.7 Landing Page Design System

A high-converting landing page follows this section hierarchy:

```
1. Hero
   - Headline: 6–12 words, benefit-focused, above the fold
   - Subheadline: 1–2 sentences expanding the headline
   - Primary CTA: high contrast, action verb + outcome ("Start free trial")
   - Social proof: logos, user count, or rating
   - Hero visual: product screenshot, illustration, or video

2. Problem / Agitate
   - Describe the pain point the audience knows
   - Make them feel understood before offering the solution

3. Solution Introduction
   - What the product/service does in plain language
```

#### Single-Speaker Production

Create a `.txt` file, then call `text_to_speech` with the desired voice.

**Available Gemini Voices (sample):**

| Voice | Character |
|---|---|
| `kore` | Default. Neutral, professional |
| `charon` | Deep, authoritative |

#### Multi-Speaker Dialogue Production

Create a `.json` file with speaker assignments:

```json
[
  {"speaker": "aoede", "text": "Welcome to the show. I'm your host, and today we're diving into brand strategy."},
  {"speaker": "charon", "text": "Thanks for having me. Brand strategy is something most founders get completely wrong."},
  {"speaker": "aoede", "text": "What's the biggest mistake you see?"},
  {"speaker": "charon", "text": "They confuse logo design with brand identity. The logo is maybe five percent of the picture."}
]
```

Then call `text_to_speech` with the `.json` file path. Voices are automatically assigned per speaker.

#### 99 Language Support

TTS supports 99 languages. Specify the language in your text content — no separate parameter needed. The model detects and matches language to appropriate phonology.

Common ISO codes for reference:
- English: `en` | Spanish: `es` | French: `fr` | German: `de` | Portuguese: `pt`
- Japanese: `ja` | Korean: `ko` | Chinese: `zh` | Arabic: `ar` | Hindi: `hi`
- Italian: `it` | Dutch: `nl` | Russian: `ru` | Turkish: `tr` | Polish: `pl`

### 6.2 Audio Transcription System

#### Configuration Options

| Parameter | Options | Notes |
|---|---|---|
| `diarize` | true / false | Enable speaker identification |
| `num_speakers` | 1–32 | Hint for diarization accuracy (leave null for auto-detect) |
| `timestamps` | `none` / `word` / `character` | `none` → plain .txt; others → .json with timing |
| `language_code` | ISO 639-1 | Auto-detected if not specified |

#### Supported Formats
`.mp3`, `.wav`, `.mp4`, `.webm`, `.m4a`, `.ogg`, `.flac`

#### Speaker Diarization Labeling Convention

When diarizing multi-speaker content, label consistently:

```
SPEAKER_00: [first speaker heard]

```

For named interviews/podcasts, rename speakers post-transcription:

```python
transcript = transcript.replace("SPEAKER_00:", "HOST:")
```

### 6.3 Podcast / Voiceover Production Workflow

```
Step 1: Script Preparation
  - Write script as multi-speaker JSON (for 2 speakers) or .txt (single voice)
  - Mark pronunciation notes in brackets: [EMPH: strategy] [PAUSE: 1s]
  - Review for natural speech patterns — written language ≠ spoken language

Step 2: Voice Selection
  - Match voice character to brand archetype (see Section 3.2)
  - Test with a 50-word sample before committing to full script
  - For multi-speaker: assign voices by character (e.g., aoede=host, charon=expert guest)

Step 3: Generation
  - Generate audio with text_to_speech tool
  - Review output MP3

Step 4: Post-Processing (if needed)
  - Trim silence: ffmpeg -i input.mp3 -af silenceremove=1:0:-50dB output.mp3
  - Normalize levels: ffmpeg -i input.mp3 -filter:a loudnorm output_normalized.mp3
  - Add background music: ffmpeg -i voice.mp3 -i music.mp3 -filter_complex "[1:a]volume=0.1[bg];[0:a][bg]amix=inputs=2:duration=first" final.mp3

Step 5: Final Export
  - Deliver as MP3 for most platforms
  - WAV for professional audio editing
  - Specific podcast platforms may require specific bitrates (Spotify: 128kbps minimum)
```

### 6.4 Transcription QA Checklist

After transcribing audio, verify:

- [ ] Speaker labels correctly assigned (spot-check 3+ transitions)
- [ ] Timestamps align with audio (test word-level timestamps at minutes 0, 5, mid, end)
- [ ] Technical terms and proper nouns spelled correctly — auto-transcription often errors on brand names, product names, acronyms
- [ ] Filler words handled consistently (`um`, `uh` — decide to keep or remove)
- [ ] Sentence boundaries make sense (auto-transcription rarely includes punctuation)
- [ ] Confidence check on ambiguous segments — listen back to any flagged sections

### 9.1 Technique Selection Guide

| Situation | Technique | Why |
|---|---|---|
| Clear, simple, well-defined task | **Zero-shot** | Fastest; works well when format is unambiguous |
| Complex task needing consistent format | **Few-shot** | Examples anchor the model's output style |
| Multi-step reasoning, logic, diagnosis | **Chain-of-Thought (CoT)** | Explicit step-by-step reduces errors |
| Open-ended problem with multiple paths | **Tree-of-Thought (ToT)** | Forces exploration before committing |
| API integration, downstream processing | **Structured Output (JSON)** | Guarantees parseable responses |
| Persistent persona, rules, constraints | **System Prompt Design** | Separates static instructions from dynamic input |
| Improving prompts themselves | **Meta-Prompting** | Turns the model into a prompt optimizer |

### 9.3 Prompt Optimization Methodology

#### A/B Scoring Framework (5 Dimensions)

Score each prompt version on a 1–5 scale across:

| Dimension | What to Evaluate |
|---|---|
| **Coherence** | Output is logical, well-structured, internally consistent |
| **Accuracy** | Factually correct / task-appropriate / no hallucinations |
| **Format compliance** | Matches requested format exactly |
| **Conciseness** | No padding, no redundancy, efficient delivery |
| **Usefulness** | A human would act on or use this output |

**Threshold:** Aim for ≥4.0 across all dimensions before considering a prompt production-ready.

#### Token Reduction Patterns

```python
# BEFORE: Verbose (312 tokens)
"""
You are a highly experienced and skilled assistant who specializes in sentiment analysis.
Your job is to carefully read the text that the user provides to you and then thoughtfully
determine whether the overall sentiment expressed in that text is positive, negative, or neutral.
Please make sure to only respond with one of these three labels and nothing else.
"""

# AFTER: Lean (28 tokens — same quality)
"""Classify sentiment as POSITIVE, NEGATIVE, or NEUTRAL. Reply with label only."""

# Savings at scale: 284 tokens × rate = significant cost reduction at volume
```

**Token reduction principles:**
1. Remove role descriptions that add no constraint ("You are a highly experienced...")
2. Replace verbose instructions with direct commands
3. Move examples to few-shot format rather than describing them
4. Eliminate redundant restatements ("Your job is to... please make sure to...")

### 9.4 Common Prompt Engineering Pitfalls

| Pitfall | Description | Fix |
|---|---|---|
| **Prompt brittleness** | Works on test cases; breaks on edge cases | Always test with empty input, very long input, unicode, code injection |
| **Instruction conflicts** | "Be concise" + "be thorough" in same prompt | Resolve contradictions; prioritize one constraint |
| **Implicit format assumptions** | Model guesses the format | Always specify format explicitly |
| **Skipping regression tests** | Every prompt edit risks breaking prior cases | Build a regression suite before any changes |
| **Optimizing wrong metric** | Low token cost matters less than high accuracy for high-stakes | Match optimization target to task stakes |
| **System prompt bloat** | 2,000-token prompts that could be 200 | Test leaner versions — quality often holds |
| **Model-specific prompts** | GPT-4-tuned prompt degrades on Claude and vice versa | Test cross-model before deploying |

```
is:reply      — include replies
is:quote      — include quote tweets
```

**Content ideation workflow:**
```
1. search_social for "#[industry] -is:retweet" (recent, original posts)
2. Identify posts with high engagement or recurring themes
3. Note language patterns audiences use (use their words, not yours)
4. Find topics competitors haven't addressed yet
5. Create content that enters the existing conversation
```
