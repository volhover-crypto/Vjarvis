# 🚨 ПРОТОКОЛ ВОССТАНОВЛЕНИЯ ДЖАРВИСА

**Для:** Александр Волховер
**Цель:** Полное восстановление функционала агента после сбоя или потери контекста.

---

## 🛑 Признаки необходимости восстановления
- Агент "забыл" базовые договоренности или имя пользователя.
- Агент перестал отвечать или выдает ошибки доступа.
- Произошел критический сбой сервера или обновление OpenClaw.

---

## 🛠 ШАГ 1: Проверка "Ядра" (Твоя часть)
Выполни эти команды в терминале сервера:

1. **Проверь целостность ядра:**
   ```bash
   ls -l /root/.openclaw/workspace/CORE_IDENTITY.md
   cat /root/.openclaw/workspace/CORE_IDENTITY.md
   ```
   *Если файла нет — это плохо. Восстанови его из бекапа или создай заново по памяти.*

2. **Проверь доступы:**
   ```bash
   python3 -c "import sqlite3; print('DB OK')" 2>/dev/null || echo "Python/DB Error"
   rclone lsd google_drive: 2>&1 | head -3
   ```

---

## 🤖 ШАГ 2: Инициализация Агента (Команда для Джарвиса)
Скопируй и отправь мне эту команду в чат:

> **/reboot_full** — (или просто напиши: "Джарвис, выполни полную инициализацию по CORE_IDENTITY.md")

**Что я сделаю после этой команды:**
1. Перечитаю `CORE_IDENTITY.md` и `USER.md`.
2. Пересоберу `SESSION_BOOTSTRAP.md`, сверив актуальные версии навыков и скриптов.
3. Проверю соединение с Groq (через прокси) и Google Drive.
4. Загружу последние записи из `~/self-improving/`, чтобы восстановить "память".
5. Отчитаюсь о статусе всех систем.

---

## 🧹 ШАГ 3: "Холодный старт" (Если Шаг 2 не помог)
Если агент ведет себя неадекватно, выполни в терминале:

```bash
# 1. Очистка временных кэшей сессии
rm -rf /root/.openclaw/agents/main/sessions/*

# 2. Перезапуск Gateway
openclaw gateway restart

# 3. Запуск новой чистой сессии
# (После этого напиши в чат: "Инициализация")
```
