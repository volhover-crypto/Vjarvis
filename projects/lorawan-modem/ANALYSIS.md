# Анализ ТЗ: LoRaWAN-модем ЛВМ-485-СОЛАР-01

**Изделие:** LoRaWAN-модем промышленного исполнения для сбора, буферизации и передачи данных от сенсоров по RS-485 с автономным питанием от солнечной панели.

---

## Общая оценка

ТЗ составлено грамотно, по структуре ГОСТ 34.602-2020. 13 страниц — компактно, но охватывает все основные разделы. Есть конкретные числовые требования, что упрощает проектирование.

---

## Сильные стороны

1. **Чёткое разделение функциональных узлов** — MCU, LoRa, RS-485, память, питание, индикация
2. **Конкретные энергетические бюджеты** — сон 50-150 мкА, активный режим 40-140 мА
3. **Три варианта LoRa-решения** — STM32WL55, STM32L072+SX1262, готовый модуль — с плюсами и минусами
4. **Рекомендуемая элементная база** — ускоряет выбор компонентов
5. **Подробные требования к документации** — BOM, Gerber, pick-and-place, сборочный чертёж
6. **Реалистичная стадийность** — от прототипа до серийного комплекта

---

## Замечания и вопросы для уточнения

### Критичные (влияют на архитектуру)

1. **Вариант LoRa не выбран** — ТЗ предлагает 3 варианта (STM32WL55, STM32L0+SX1262, модуль). Для серийного изделия это ключевое решение, определяющее всю схемотехнику. Рекомендую на этапе согласования определить один базовый вариант.

2. **Тип АКБ не фиксирован** — LiFePO4 1S/2S или SLA? Это влияет на:
   - Напряжение шины питания (3.2В×n для LiFePO4, 12В для SLA)
   - Выбор контроллера заряда
   - Конфигурацию DC/DC-преобразователей
   - MPPT vs PWM для солнечной панели

3. **Время автономной работы без солнца** — не указано. Критично для расчёта ёмкости АКБ и мощности солнечной панели. Для Крыма (зима: 2-3 пасмурных дня подряд) — должно быть минимум 3-5 суток.

4. **Профиль трафика LoRaWAN** — не определён:
   - Сколько датчиков? Как часто опрос?
   - Объём данных за сеанс?
   - DR (Data Rate) — при DR0 максимум 51 байт, этого может не хватить для пакета с 16 датчиками

### Важные (влияют на конструкцию)

5. **Степень защиты** — ТЗ говорит IP65 для корпуса, но не определяет:
   - Тип гермовводов для RS-485 и солнечной панели
   - Способ крепления антенны (SMA гермоввод vs N-разъём)
   - Материал корпуса

6. **Климатический диапазон** — "-40…+60°C" — это широкий диапазон. Влияет на:
   - Выбор электролитических конденсаторов (или отказ от них)
   - Химию АКБ (LiFePO4 деградирует при -20°C, SLA лучше держит холод)
   - Выбор паяльных материалов

7. **Сервисный интерфейс** — UART bootloader/SWD/JTAG — нужен разъём внутри герметичного корпуса. Как? Через разъём с крышкой? Беспроводная диагностика?

### Мелкие

8. **Размеры платы не заданы** — только "единый электронный узел на одной плате". Для проектирования нужны габариты.

9. **Количество слоёв PCB** — "4 слоя рекомендуется, 2 допускаются". Для LoRa-диапазона 4 слоя — однозначно лучше (контроль импеданса, земляная плоскость).

10. **MicroSD** — ТЗ допускает "только при специальном требовании". Для будущего — предусмотреть на PCB контактную площадку, но не ставить разъём.

## Рекомендации по элементной базе

### LoRa: STM32WL55JC (вариант A) — предпочтительный

- **MCU + LoRa в одном корпусе** — меньше BOM, меньше площадь
- **Частота:** 150-960 МГц (покрывает RU864)
- **Потребление:** 1.8 мкА в Shutdown, 24 мВт при передаче +17 dBm
- **Класс LoRaWAN:** A/C из коробки
- **Минус:** RF-проектирование сложнее, нужен опыт

**Альтернатива:** RAK3172 (STM32WLE5 + сертификация) — если нужна быстрая сертификация

### Контроллер заряда: BQ24650 (TI)

- MPPT-алгоритм для LiFePO4
- Вход до 28В (для панели 18В)
- Программирование параметров через I2C

**Альтернатива (бюджетная):** CN3791 — проще, но без MPPT

### RS-485: ADM2587E

- Изолированный трансивер в одном корпусе
- 500 кбит/с, ESD-защита ±15 кВ
- Гальваническая изоляция 2.5 кВ

### Память: W25Q64 (Winbond)

- SPI NOR Flash, 64 Мбит (8 МБ)
- 230 мА при записи, 1 мкА в standby
- Ресурс: 100K циклем записи

Терминатор RS-485: Bourns SM712 (TVS), 120 Ом терминальный резистор
