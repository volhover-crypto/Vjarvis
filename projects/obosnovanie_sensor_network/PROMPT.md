# Технико-экономическое обоснование: распределённая сеть датчиков в агрохозяйстве

## Промпт для нейросети

```
Роль: Ты — эксперт по точному земледелию и IoT-системам в сельском хозяйстве с 15+ летним опытом.

Инструкция: Подготовь развёрнутое технико-экономическое обоснование внедрения распределённой сети датчиков в агрохозяйстве.

Важно: Если для качественного обоснования не хватает входных данных (площадь хозяйства, культуры, регион, текущее состояние инфраструктуры) — задай уточняющие вопросы ДО начала работы. Не выдумывай параметры — лучше спроси.

Шаги:
1. Опиши архитектуру распределённой сети датчиков: типы датчиков, топология сети, протоколы передачи данных, узлы сбора и обработки.
2. Определи ключевые измеряемые параметры (почва, воздух, растение, осадки) и обоснуй выбор пороговых значений.
3. Опиши механизм обработки данных: от сырых показаний до управляющих решений (полив, подкормка, защита).
4. Проведи экономический расчёт: CAPEX, OPEX, ожидаемая экономия (вода, удобрения, урожайность), срок окупаемости.
5. Оцени риски и ограничения внедрения.

Конечная цель: Документ, на основании которого руководитель хозяйства принимает решение о внедрении. Формат — профессиональный отчёт с цифрами.

Ограничения: Не абстрактные рассуждения — только конкретные цифры, примеры, расчёты. Объём — 1500–2500 слов.
```

## Инструкция по использованию

1. Скопируй промпт выше
2. Вставь в любую LLM (Claude, GPT, Gemini, GigaChat, YandexGPT)
3. Если нейросеть задаст уточняющие вопросы — ответь для твоего хозяйства
4. Полученный результат сохрани как файл `obosnovanie_sensor_network.md`
5. Отправь файл на почту: volhover@yandex.ru

## Что включить в ответы на уточняющие вопросы

- Площадь хозяйства (га)
- Основные культуры
- Регион / климатическая зона
- Текущий способ полива (капельный, дождевание, нет)
- Есть ли метеостанция
- Бюджет (примерный)
