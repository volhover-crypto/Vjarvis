# Технико-экономическое обоснование внедрения распределённой сети датчиков в агрохозяйстве

**Площадь:** 1500 га  
**Культуры:** виноград, косточковые, питомники, овощи  
**Регион:** Крым / Юг России  
**Полив:** варьируется (капельный → отсутствует)  
**Метеостанции:** отсутствуют или единичные  
**Бюджет:** определяется экономическим эффектом

---

## 1. Архитектура распределённой сети датчиков

### 1.1. Топология сети

Предлагается трёхуровневая архитектура:

**Уровень 1 — Полевые датчики (сенсорный слой)**
- Почвенные датчики: влажность (ёмкостные/тензиометрические), EC, температура почвы
- Метеодатчики: T воздуха, RH, осадки (радарные), ветер, солнечная радиация
- Фитодатчики: диаметр ствола (дендрометры), температура листа, поток сока
- Для винограда: датчики диаметра ягоды (линейные LVDT)

**Уровень 2 — Шлюзы и контроллеры (edge-слой)**
- Шлюзы LoRaWAN / NB-IoT на каждые 50–100 га
- Edge-контроллеры для локальной обработки и принятия решений в реальном времени
- Питание: солнечные панели + аккумуляторы (автономность 5+ лет)

**Уровень 3 — Центральный сервер (cloud/локальный)**
- Сервер сбора, хранения и анализа данных (PostgreSQL + TimescaleDB)
- Веб-интерфейс визуализации и управления
- API для интеграции с системами полива, подкормки, защиты

### 1.2. Протоколы передачи данных

| Протокол | Применение | Дальность | Потребление |
|----------|------------|-----------|-------------|
| LoRaWAN | Почвенные и метеодатчики | до 15 км | ~50 мВт |
| NB-IoT | Шлюзы → сервер | покрытие сотовой сети | ~200 мВт |
| RS-485/Modbus | Промышленные контроллеры | до 1.2 км | проводной |
| Wi-Fi | Edge-контроллеры → локальная сеть | до 300 м | ~1 Вт |

### 1.3. Плотность размещения датчиков

| Культура | Почвенные (на га) | Метео (на 100 га) | Фитодатчики (на га) |
|----------|-------------------|-------------------|---------------------|
| Виноград | 3–5 | 1 | 2–3 |
| Косточковые | 2–3 | 1 | 1–2 |
| Питомники | 5–8 | 1 | 3–5 |
| Овощи | 4–6 | 1 | 1–2 |

## 3. Механизм обработки данных

### 3.1. Конвейер данных

```
Сырые показания → Валидация → Агрегация → Анализ → Решение → Действие
```

**Этап 1. Валидация (edge, <1 сек)**
- Отсечение выбросов (нули, зависания, выход за физические пределы)
- Интерполяция пропусков (<5% — линейная, >5% — флаг «данные ненадёжны»)

**Этап 2. Агрегация (edge, каждые 15 мин)**
- Средние, мин, макс за интервал
- Расчёт производных (скорость изменения влажности, GDD, ET₀)

**Этап 3. Анализ (сервер, каждые 30 мин)**
- Сравнение с порогами для каждой культуры и фенофазы
- Расчёт SWD (Soil Water Deficit) по балансовой модели
- Прогноз рисков болезней (милдью, оидиум, серая гниль) по погодным индексам
- Расчёт GDD для прогноза фенофаз

**Этап 4. Решение (сервер, каждые 1–4 часа)**
- Формирование рекомендаций: полив (объём, время), подкормка (тип, доза), защита (препарат, срок)
- Приоритизация по критичности: красный → жёлтый → зелёный

**Этап 5. Действие**
- Автоматическое: включение/выключение клапанов полива, запуск фертигации
- Ручное (с подтверждением): обработка фунгицидами, внесение удобрений

### 3.2. Модели принятия решений

**Полив (для всех культур с системой полива):**
- SWD-модель: полив при достижении порога дефицита влаги
- ETc-модель: ETc = ET₀ × Kc (коэффициент культуры по фенофазе)
- Для винограда: регулируемый дефицитный полив (RDI) — стресс в определённые фазы для качества ягод

**Защита от болезней:**
- Милдью (Plasmopara viticola): индекс Гравотти — обработка при T>10°C, RH>90%, осадки >10 мм
- Оидиум (Erysiphe necator): индекс Шарпа — обработка при T 20–28°C, RH 50–70%
- Серая гниль (Botrytis): обработка при T 15–25°C, RH>85% за 48 часов до сбора

**Подкормка:**
- Привязка к фенофазам (BBCH) и данным листовой диагностики
- Дозирование по EC почвенного раствора

## 5. Риски и ограничения

### 5.1. Технические риски

| Риск | Вероятность | Влияние | Митигация |
|------|-------------|---------|-----------|
| Выход из строя датчиков (коррозия, механика) | Высокая | Среднее | Замена 10%/год в OPEX, мониторинг состояния |
| Потеря связи (LoRaWAN/NB-IoT) | Средняя | Высокое | Edge-автономность, локальные решения без сервера |
| Неточность датчиков (дрейф EC, влажности) | Средняя | Среднее | Калибровка 2 раза/год, кросс-валидация |
| Кибербезопасность | Низкая | Высокое | Шифрование, VPN, изоляция сети |

### 5.2. Организационные риски

| Риск | Митигация |
|------|-----------|
| Сопротивление персонала | Обучение, демонстрация результатов на пилотном участке |
| Нехватка квалифицированных кадров | Аутсорс обслуживания, удалённая поддержка вендора |
| Зависимость от вендора | Открытые протоколols (LoRaWAN, Modbus), возможность замены компонентов |

### 5.3. Ограничения

- На участках без полива эффект ограничивается мониторингом и защитой (нет экономии воды)
- Питомники требуют наибольшей плотности датчиков — высокая стоимость на га
- Овощи с коротким циклом требуют быстрой окупаемости — пилотный запуск обязателен
- Отсутствие метеостанций компенсируется установкой 15 шлюзов с метеоблоком

## 6. Рекомендации по внедрению

### Фаза 1 — Пилот (3–6 месяцев), 100–200 га
- Виноградник + овощной участок
- 1 метеостанция, 50 почвенных датчиков, 10 фитодатчиков
- Оценка реального экономического эффекта
- Обучение персонала

### Фаза 2 — Масштабирование (6–12 месяцев)
- Оставшиеся виноградники и косточковые
- Полный контур мониторинга и управления

### Фаза 3 — Полное покрытие (12–18 месяцев)
- Питомники, овощи, участки без полива (мониторинг + защита)
- Интеграция с ERP/учётом хозяйства

## 7. Выводы

1. **Срок окупаемости 1–4 месяца** в зависимости от культуры и наличия полива
2. **Основной драйвер эффекта** — прирост урожайности (12–20%) за счёт точного управления
3. **Критическая культура для внедрения** — виноград (наибольшая маржинальность, чувствительность к водному режиму)
4. **Участки без полива** — эффект от мониторинга болезней и прогноза фенофаз (снижение потерь 10–15%)
5. **Пилотный запуск обязателен** — для валидации моделей и обучения персонала перед масштабированием
